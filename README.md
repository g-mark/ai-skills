# ai-skills

## <img src="layered-design/assets/layered-design-small.png" height="90" /> layered-design

An attempt at directing an AI towards thinking in layers, roughly analogous to using a C4 model: start with the _context_, define the _containers_, identify the _components_, write the _code_.

I want to be able to read (and think about) a design in layers, only considering details when relevant. Most of the design skills I've tried leak details upwards: proudly presenting code very early on. I would much rather leave the code out entirely or only have it in the leaf-node bullet points.

This skill, as-is, _does_ produce some code, but it should do so mostly for interfaces and/or important algorithms.

I recommend using high-thinking effort with the best model you have access to for the best results with this skill.
